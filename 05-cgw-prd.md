## 产品定位

CGW 是一套**长连接网关系统**，专门用于维护和管理持久化连接，支持高并发长连接场景。

**核心价值：** 统一管理各类长连接，降低连接维护成本，提升消息推送效率。

---

## 应用场景

### 1. 即时通讯（IM）
- 聊天消息推送
- 在线状态同步
- 群组消息广播
- 输入状态通知

### 2. 消息推送
- App 推送通知
- 系统公告
- 运营消息
- 实时告警

### 3. 实时数据同步
- 协同编辑
- 实时看板
- 股票行情
- 游戏状态同步

### 4. 物联网（IoT）
- 设备状态上报
- 远程控制指令
- 固件升级通知
- 设备告警

## 协议支持矩阵

| 场景 | 协议 | 说明 | 优先级 |
|------|------|------|:------:|
| 网页端长连 | **WS / WSS** | WebSocket，浏览器原生支持 | P0 |
| 网页端备选 | **SSE** | Server-Sent Events，HTTP 降级方案 | P1 |
| 移动端/App | **TCP 自定义** | 移动网络优化，心跳保活 | P0 |
| 移动端备选 | **MQTT** | 低功耗、弱网适配 | P1 |
| 移动端备选 | **gRPC** | 双向流，Protobuf 序列化 | P1 |
| 微服务内部 | **gRPC** | 服务间长连接通信 | P0 |
| 微服务备选 | **HTTP/2** | 多路复用 | P1 |
| 物联网设备 | **MQTT** | 低功耗、发布订阅模式 | P0 |
| 物联网备选 | **TCP** | 自定义协议，最小开销 | P1 |

## 架构设计

### 整体架构

```
                    客户端（Web/App/IoT）
                           │
                           ▼
        ┌──────────────────────────────────────┐
        │           接入层（LB）                 │
        │        L4/L7 负载均衡                 │
        └──────────────────┬───────────────────┘
                           │
          ┌────────────────┼────────────────┐
          ▼                ▼                ▼
    ┌─────────┐      ┌─────────┐      ┌─────────┐
    │  CGW-1  │      │  CGW-2  │      │  CGW-3  │
    │ WS/TCP  │      │ WS/TCP  │      │ WS/TCP  │
    └────┬────┘      └────┬────┘      └────┬────┘
         │                │                │
         └────────────────┼────────────────┘
                          │
                          ▼
        ┌──────────────────────────────────────┐
        │           消息路由层                   │
        │     Message Router (Redis/Kafka)     │
        └──────────────────┬───────────────────┘
                           │
                          ▼
        ┌──────────────────────────────────────┐
        │           业务服务层                   │
        │    IM Service / Push Service / ...   │
        └──────────────────────────────────────┘
```

### 数据面
- **语言**：Rust（高性能、异步）
- **框架**：Tokio（异步运行时）
- **协议**：WebSocket、TCP、MQTT、gRPC

### 控制面
- **语言**：Go
- **存储**：Redis（会话状态）、PostgreSQL（持久化）
- **消息队列**：Kafka（消息路由）

## 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 异步运行时 | Tokio | Rust 生态最成熟 |
| WebSocket | tokio-tungstenite | 高性能 WS 实现 |
| TCP | tokio-net | 异步 TCP |
| MQTT | rumqttc | Rust MQTT 客户端 |
| gRPC | tonic | Rust gRPC 框架 |
| 会话存储 | Redis Cluster | 高性能、分布式 |
| 消息队列 | Kafka | 高吞吐、持久化 |
| 服务发现 | Consul / Nacos | 动态服务注册 |

## 与其他网关的关系

| 网关 | 定位 | 与 CGW 的关系 |
|------|------|--------------|
| **UGW** | 统一网关（HTTP 为主） | 互补，CGW 专注长连接 |
| **AGW** | 入站 API 网关 | 短连接请求走 AGW |
| **EGW** | 出站代理网关 | 出站请求走 EGW |

**协作模式：**
```
客户端 ──短连接──→ AGW（UGW 反向代理）──→ 业务服务
   │
   └──长连接──→ CGW ──→ 消息推送服务
```
