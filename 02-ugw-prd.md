## 产品定位

UGW 是**统一网关**的总称，包含三个子网关：

| 子网关 | 全称 | 定位 |
|--------|------|------|
| **AGW** | Agent Gateway | 智能体网关（入站/反向代理） |
| **EGW** | Egress Gateway | 出口网关（出站/正向代理） |
| **CGW** | Connection Gateway | 长连接网关（持久连接） |

**核心理念：** 一套架构，三种场景，统一管理。

```
                    ┌─────────────────────────────────┐
                    │         UGW 统一网关             │
                    │      (Unified Gateway)          │
                    └─────────────────────────────────┘
                                    │
            ┌───────────────────────┼───────────────────────┐
            ▼                       ▼                       ▼
    ┌───────────────┐       ┌───────────────┐       ┌───────────────┐
    │      AGW      │       │      EGW      │       │      CGW      │
    │  Agent Gateway │       │Egress Gateway │       │Connection GW  │
    │   入站/反向代理  │       │  出站/正向代理  │       │   长连接管理   │
    └───────────────┘       └───────────────┘       └───────────────┘
            │                       │                       │
            ▼                       ▼                       ▼
       内网服务               互联网出口              消息推送/IM
```

### 整体架构

```
                    ┌─────────────────────────────────┐

                    │  ┌─────────┐   ┌─────────┐     │

                    │         ┌────┴────┐             │
```

### 控制面（Control Plane）
- **语言**：Go
- **存储**：PostgreSQL + Redis
- **特点**：配置管理、策略下发、监控

## 核心设计：配置驱动模式

### 模式判断逻辑

```
启动时读取配置
       ↓
  mode: reverse  →  反向代理模式（AGW）
  mode: forward  →  正向代理模式（EGW）
  mode: dual     →  双模式（同时支持）
```

### 配置示例

```yaml
# 反向代理模式（AGW）
ugw:
  mode: reverse
  listen:
    - port: 443
      protocol: https
  upstream:
    - name: backend-service
      addr: 10.0.0.1:8080

# 正向代理模式（EGW）
ugw:
  mode: forward
  listen:
    - port: 3128
      protocol: http
  egress:
    - name: internet-access
      strategy: random  # random / fixed / rotation

# 双模式
ugw:
  mode: dual
  reverse:
    listen: 443
  forward:
    listen: 3128
```

## 网关术语对照

### 常见网关类型

| 术语 | 英文 | 说明 |
|------|------|------|
| 长连接网关 | Connection Gateway | 维护持久连接 |
| 推送网关 | Push Gateway | 消息推送服务 |
| IM 网关 | IM Gateway | 即时通讯接入 |
| 接入网关 | Access Gateway | 统一接入层 |
| 消息路由网关 | Message Router | 消息分发路由 |
| TCP/WS/MQTT 接入层 | Protocol Gateway | 协议适配层 |

### 连接类型与协议

| 场景 | 协议 | 说明 |
|------|------|------|
| 网页端长连 | WS / WSS / SSE | WebSocket、Server-Sent Events |
| 移动端/App 长连 | TCP 自定义、MQTT、gRPC | 移动网络优化 |
| 微服务内部长连 | gRPC、HTTP/2 | 服务间通信 |
| 物联网设备 | MQTT、TCP | 低功耗、弱网适配 |
