## 产品定位

AGW 是一套面向**智能体（Agent）的网关系统**，用于管理和路由 AI Agent 的请求、流量控制、认证鉴权等。

---

## 核心功能

### 1. 请求路由
- 智能路由到不同的 Agent
- 按意图/领域/版本分发
- 动态权重调整

### 2. 负载均衡
- 多实例流量分配
- 健康检查 + 自动摘除
- 水平扩容支持

### 2.1 流量调度策略 🎯
**基于身份识别：**
- [ ] **IP 哈希** — 未登录用户，按客户端 IP 哈希
- [ ] **用户 ID 哈希** — 已登录用户，按用户 ID 哈希
- [ ] **设备 ID 哈希** — 按设备指纹哈希
- [ ] **Cookie/Session 哈希** — 按会话 ID 哈希

**基于地理位置：**
- [ ] **国家/区域调度** — 按用户所在国家/区域路由
- [ ] **就近路由** — 路由到最近的数据中心
- [ ] **运营商调度** — 按运营商（电信/联通/移动）优化

**基于属性：**
- [ ] **租户隔离** — 按租户 ID 隔离到专属资源池
- [ ] **版本路由** — 按 App 版本路由到不同后端
- [ ] **端类型路由** — Web / App / API 分开处理

**调度算法：**
- [ ] 轮询 (Round Robin)
- [ ] 加权轮询 (Weighted Round Robin)
- [ ] 最少连接 (Least Connections)
- [ ] IP 哈希 (Consistent Hashing)
- [ ] 随机 (Random)

## 用户场景

### 1. 企业 AI Agent 部署
- 企业内部多个 AI Agent（客服、代码助手、数据分析）
- 统一入口、统一鉴权、统一监控
- 防止 API Key 泄露

### 2. 多租户 SaaS 平台
- 多个客户共享一套 Agent 服务
- 资源隔离、配额控制
- 按用量计费

### 3. Agent 开发测试
- 开发阶段快速切换不同 Agent
- 灰度发布 A/B Testing
- 回滚和流量控制

### 4. 成本控制场景
- 公司有多人用 AI，需要统一管理 API 消耗
- 限流防止某个用户消耗过多
- 统计各部门/项目的用量

### 5. 安全合规场景
- 过滤敏感输入/输出
- 审计日志（谁在什么时间调用了什么）
- 数据脱敏

## 技术方案

### 架构设计

**数据面（Data Plane）**
- 语言：Rust
- 职责：高性能请求处理、限流、熔断、缓存
- 目标：极致性能和低延迟

**控制面（Control Plane）**
- 语言：Go
- 数据库：PostgreSQL
- 缓存：Redis
- 职责：配置管理、路由规则、认证鉴权、日志聚合、API 网关

### 技术栈

```
数据面：Rust + Tokio
控制面：Go + Gin + PostgreSQL + Redis
通信：gRPC（控制面 ↔ 数据面）
部署：Docker/Kubernetes
```

### 数据面 Rust 职责
- 请求转发
- 限流/熔断
- 请求缓存
- 连接池管理

### 控制面 Go 职责
- 配置下发
- 路由规则管理
- 认证鉴权
- 日志/监控聚合
- 用户/租户管理
